## 💡 快速答案

### Q: 如何实现一个新功能？

**A**: 5步搞定
1. 创建切片目录 `Modules/{Module}/{Feature}/`
2. 定义 Command（`{Feature}.cs`）
3. 实现 Handler（`{Feature}Handler.cs`）
4. 添加 Endpoint（`{Feature}Endpoint.cs`）
5. 运行测试

详见: [Wolverine快速上手指南 - 1.1节](./Wolverine快速上手指南.md#11-从零实现一个功能)

### Q: Handler 如何自动注入依赖？

**A**: Wolverine 约定：第一个参数是消息，后续参数自动注入

```csharp
public async Task<TResult> Handle(
    TCommand command,           // 第一个：消息
    IDocumentSession session,   // 自动注入
    ILogger logger,             // 自动注入
    CancellationToken ct)
```

详见: [Wolverine快速上手指南 - 2.1节](./Wolverine快速上手指南.md#21-wolverine-的-3-个核心约定)

### Q: 如何跨模块调用？

**A**: 使用 `IMessageBus`
- 同步: `await bus.InvokeAsync(command)`
- 异步: `await bus.PublishAsync(event)`

详见: [Wolverine模块化架构蓝图 - 6.1节](./Wolverine模块化架构蓝图.md#61-通信方式选择)

### Q: 如何实现事务？

**A**: 使用 `[Transactional]` 特性或全局策略

```csharp
[Transactional]
public async Task Handle(Command cmd, IDocumentSession session) { }
```

详见: [Wolverine快速上手指南 - 3.5节](./Wolverine快速上手指南.md#35-场景-5添加事务支持)

### Q: 如何调试 Handler 未被发现？

**A**: 检查3点
1. 方法名是 `Handle` 或 `HandleAsync`
2. Handler 类是 `public`
3. Handler 所在程序集被扫描

详见: [Wolverine快速上手指南 - 5.1节](./Wolverine快速上手指南.md#错误-1handler-未被发现)
