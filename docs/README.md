## 📖 阅读建议

### 新加入的开发者

**第一天：**
1. 阅读 [架构原则与约束.md](./架构原则与约束.md) 的"核心架构决策"和"模块隔离原则"
2. 理解 3 条铁律（绝对不能违反）

**第二天：**
1. 阅读 [MVP开发计划.md](./MVP开发计划.md) 的当前周任务
2. 看懂"垂直切片清单"和"跨模块事件流"

**第三天：**
1. 运行架构测试：`dotnet test ./src/tests/ArchitectureTests/`
2. 阅读一个已实现的垂直切片代码（如果有）

### 架构 Review 时

**检查清单：**
- [ ] 是否违反了 3 条铁律？
- [ ] 跨模块通信是否用了事件？
- [ ] 模块内是否出现了 Repository/Service 命名？
- [ ] Handler 签名是否符合约定？
- [ ] 架构测试是否通过？
