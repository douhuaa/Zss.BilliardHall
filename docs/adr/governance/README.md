# 治理与流程层 ADR (ADR-900 ~ 999)

**编号范围**：ADR-900 ~ ADR-999  
**层级定位**：架构治理 / 流程约束 / 破例管理  

---

## 定义

治理与流程层 ADR 定义架构的自动化校验、破例流程、变更管理等治理机制：

- **架构测试规范**：如何通过自动化测试保障架构约束
- **ADR 流程管理**：ADR 的新增、修订、废弃流程
- **破例与偿还**：如何申请破例、记录破例、归还技术债
- **变更审批流程**：不同级别变更的审批权限和流程

---

---

## 与宪法层的关系

治理层为宪法层提供执行保障：

```
宪法层（WHAT）
    ↓
治理层（HOW TO ENFORCE）
    ↓
自动化测试 + CI/CD + 人工审批
```

治理层 ADR 确保宪法层约束能够：
- ✅ 自动化检测（架构测试）
- ✅ CI 阻断（测试失败 = 构建失败）
- ✅ 显式破例（记录在案、可追溯）
- ✅ 定期审计（归还技术债）
