# ADR-990：文档演进路线图

> ⚖️ **本 ADR 是文档体系演进路线图的唯一管理标准，定义路线图的结构、更新机制和与 ADR 的关联。**

**状态**：✅ Accepted  
## Focus（聚焦内容）

- 路线图结构与组织
- 季度更新机制
- 与 ADR/RFC 的关联
- 状态追踪
- 透明度与可访问性

### ADR-990_2：季度更新机制（Rule）

#### ADR-990_2_1 必须每季度更新路线图
**必须**每季度更新路线图。

#### ADR-990_2_2 更新时机
**更新时机**：
- 每季度末（3月、6月、9月、12月）
- 重大变更发生时

#### ADR-990_2_3 更新内容
**更新内容**：
- 完成项目状态更新
- 新项目添加
- 进度更新
- 优先级调整
- 指标更新

#### ADR-990_2_4 评审流程
**评审流程**：
1. 文档团队内部评审
2. 架构委员会审核
3. 社区反馈收集（2周）
4. 最终发布

#### ADR-990_2_5 核心原则
**核心原则**：
> 定期更新，及时反映，持续改进。

**判定**：
- ❌ 路线图过时
- ❌ 更新不及时
- ✅ 季度更新，内容最新

### ADR-990_3：与 ADR/RFC 的关联（Rule）

#### ADR-990_3_1 路线图项目必须与 ADR/RFC 关联
路线图项目 **必须**与 ADR/RFC 关联。

#### ADR-990_3_2 关联方式
**关联方式**：
- 每个项目必须链接到相关 ADR 或 RFC
- 格式：`关联 ADR：ADR-XXX` 或 `关联 RFC：RFC-XXX`

#### ADR-990_3_3 状态同步
**状态同步**：
- ADR 状态变更时，路线图自动更新
- RFC 采纳时，自动添加到路线图

#### ADR-990_3_4 核心原则
**核心原则**：
> 关联明确，状态同步，追踪可行。

**判定**：
- ❌ 项目无 ADR 关联
- ❌ 状态不同步
- ✅ 关联明确，同步更新

#### ADR-990_3_5 最低自动化要求（Minimum Automation Requirement）

在 ADR-990 生效后：

1. **必须**在 1 个季度内提供以下最小能力之一：
  - 路线图结构校验脚本（markdown schema / regex）
  - 路线图与 ADR 关联完整性检查

2. 在自动化未完成前：
  - 路线图更新 **仍然是强制人工审查项**
  - Guardian Agent 必须在 PR 中显式给出警告

3. 自动化能力缺失 **不得作为跳过治理的理由**

### ADR-990_4：状态追踪（Rule）

#### ADR-990_4_1 必须实时追踪项目状态
**必须**实时追踪项目状态。

#### ADR-990_4_2 状态定义
**状态定义**：
- 🟢 **完成**：项目已完成并交付
- 🟡 **进行中**：项目正在开发中
- 🔴 **延迟**：项目超出预计时间
- 🔵 **规划中**：项目已规划但未开始
- ⚪ **待开始**：项目已批准但未规划

#### ADR-990_4_3 更新机制
**更新机制**：
- 项目负责人每周更新状态
- 自动化脚本检查进度
- 延迟项目自动标记

#### ADR-990_4_4 核心原则
**核心原则**：
> 状态清晰，更新及时，进度可见。

**判定**：
- ❌ 状态不明
- ❌ 更新滞后
- ✅ 实时追踪，清晰可见

### ADR-990_5：透明度与可访问性（Rule）

#### ADR-990_5_1 路线图必须公开可访问
路线图 **必须**公开可访问。

#### ADR-990_5_2 访问方式
**访问方式**：
- GitHub 仓库中的 `docs/ROADMAP.md`
- 项目网站集成
- 定期邮件通知

#### ADR-990_5_3 反馈机制
**反馈机制**：
- GitHub Issue Template：`roadmap-feedback`
- 季度评审会议
- 匿名反馈表单

#### ADR-990_5_4 核心原则
**核心原则**：
> 公开透明，易于访问，积极反馈。

**判定**：
- ❌ 路线图不公开
- ❌ 缺乏反馈渠道
- ✅ 公开可访问，反馈机制完善

### ADR-990_6：路线图失效与阻断机制（Rule）

#### ADR-990_6_1 路线图过期视为治理失效

若 `docs/ROADMAP.md` 的 **最后更新日期** 超过当前季度末 + 7 天，
则视为 **文档治理失效状态**。

#### ADR-990_6_2 失效后的强制措施

当路线图处于失效状态时：

1. ❌ 禁止合并任何 **文档治理类 ADR**
2. ❌ 禁止新增 **Documentation / Governance 类 RFC**
3. CI 必须产生失败状态，提示：
   > “ROADMAP 未更新，文档治理被冻结”

#### ADR-990_6_3 恢复条件

仅当：
- 路线图完成季度更新
- 架构委员会确认并合并

治理冻结状态方可解除。

## Prohibited（禁止行为）

#### ADR-990_P_1 禁止无责任人的路线图项目

任何路线图项目 **必须**明确负责人（GitHub 用户或角色）。
缺失负责人视为无效项目。

#### ADR-990_P_2 禁止无 ADR / RFC 关联的项目

路线图项目若无法关联 ADR 或 RFC：
- 不得进入“进行中”
- 不得占用季度目标位置

#### ADR-990_P_3 禁止永久“规划中”状态

项目在“规划中”状态超过 1 个季度，
必须：
- 转为“延迟”并说明原因
- 或从路线图中移除
