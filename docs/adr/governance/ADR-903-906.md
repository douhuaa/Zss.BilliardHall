# ADR-903-906：ArchitectureTests 执法治理体系

> ⚖️ **本 ADR 链整合 ArchitectureTests 的命名、组织、最小断言及 CI / Analyzer 映射规则，实现完整的自动裁决闭环。**

---

## Focus（聚焦内容）

本体系覆盖以下治理目标：

1. **命名与组织规范（ADR-903）**  
   - 测试类、方法与项目命名必须明确映射 ADR  
   - 测试目录结构与 ADR 编号一一对应  
   - 禁止跨 ADR 混合测试或空弱测试

2. **最小断言语义（ADR-904）**  
   - 每条 ArchitectureTest 至少包含 1 个有效断言  
   - 测试方法映射单一 ADR 子规则  
   - 弱断言或形式化断言禁止

3. **Analyzer / CI Gate 映射协议（ADR-906）**  
   - 所有 ArchitectureTests 必须自动注册至 CI / Analyzer  
   - 测试失败直接映射 ADR 子规则，支持 L1/L2 执行等级  
   - 支持破例与偿还机制  
   - ADR 生命周期变更自动同步

**目标**：让 ArchitectureTests 成为 ADR 的“可执行镜像”，支持自动化裁决与反作弊机制。

## Decision（裁决）

### ADR-903：命名与组织规范

1. **ArchitectureTests 必须集中在独立项目**  
   - `<SolutionName>.Tests.Architecture`  
   - 禁止混合单元或集成测试项目  
2. **测试目录按 ADR 编号分组**  
   - 每个 ADR 对应一个目录 `/ADR-XXXX/`  
   - 不存在 ADR 的目录非法  
3. **禁止跨 ADR 混合测试**  
   - 单个测试类或文件仅覆盖一个 ADR  
4. **测试类命名必须显式绑定 ADR**  
   - 格式：`ADR_<编号>_Architecture_Tests`  
5. **测试方法必须映射 ADR 子规则**  
   - 格式：`ADR_<编号>_<子规则>_<行为描述>`  
6. **测试失败信息必须包含 ADR 编号**  
7. **ArchitectureTests 不得为空或弱断言**  
8. **不得跳过或条件禁用测试（除破例）**

### ADR-904：最小断言语义规范

1. 每个测试类至少包含 1 个有效断言  
2. 每个测试方法只能映射一个 ADR 子规则  
3. 所有断言失败信息必须可反向溯源到 ADR  
4. 禁止弱断言或形式化断言（如 `Assert.True(true)`）

### ADR-906：Analyzer / CI Gate 映射协议

1. 所有 ArchitectureTests 必须注册至 CI / Analyzer  
2. 测试执行失败必须映射 ADR 子规则  
3. 支持 L1/L2 执行等级  
   - L1 失败阻断 CI / 部署  
   - L2 失败记录警告，人工 Code Review  
4. 破例与偿还机制必须自动记录，包含 ADR 编号、测试类/方法、理由、到期时间  
5. CI / Analyzer 必须检测空测试、弱断言及多 ADR 覆盖情况  
6. ADR 生命周期变更同步：废弃或替代的 ADR 自动标记测试为 Obsolete 或删除

## Non-Goals（明确不管什么）

- 架构规则本身设计
- 测试框架或断言库选型
- 性能优化或测试速度
- 业务逻辑正确性

---

## Prohibited（禁止行为）

- 无 ADR 编号的 ArchitectureTests  
- 一个测试类覆盖多个 ADR  
- 弱断言或空测试未被检测  
- 跳过 CI / Analyzer 阻断  
- 弃用 ADR 的测试未同步处理
