# 静态结构层 ADR (ADR-100 ~ 199)

**编号范围**：ADR-100 ~ ADR-199  
**层级定位**：模块组织 / 静态边界 / 目录结构

---

## 定义

静态结构层 ADR 定义系统的组织结构、模块划分、命名规范等静态约束：

- **模块划分策略**：如何将业务能力组织为模块
- **目录结构规范**：源码目录的组织方式
- **静态边界规则**：模块之间的依赖约束
- **命名规范细则**：类型、文件、项目的命名规则

---

## 与宪法层的关系

静态结构层是对宪法层的细化和补充：

```
宪法层定义原则：
- ADR-0001：模块隔离原则
- ADR-0002：三层装配模型
- ADR-0003：命名空间规范

↓ 细化

结构层定义细节：
- ADR-1xx：具体模块划分策略
- ADR-1xx：具体目录组织细则
- ADR-1xx：具体命名规范扩展
```

## 当前状态

当前项目的静态结构约束主要由宪法层 ADR 定义：

- ADR-0001：模块组织原则
- ADR-0002：三层启动体系
- ADR-0003：命名空间规范

结构层细化规范：

- **ADR-120**：[领域事件命名规范](ADR-120-domain-event-naming-convention.md) - 定义领域事件的命名规则、命名空间组织和版本演进策略
- **ADR-121**：[契约（Contract）与 DTO 命名组织规范](ADR-121-contract-dto-naming-organization.md) - 定义跨模块数据传递的契约命名、目录组织、版本管理和约束规范
