## 🏗️ 核心原则

### 1. 单一真相源（SSOT）

> **定义**: 每个知识点只在一个权威文档中详细说明，其他地方只能引用

**示例**:

| 主题 | 唯一权威文档 |
|------|-------------|
| Wolverine 架构 | `03_系统架构设计/Wolverine模块化架构蓝图.md` |
| 代码风格 | `06_开发规范/代码风格.md` |
| Saga 使用 | `06_开发规范/Saga使用指南.md` |

✅ **正确**: `详见 [Saga 使用指南](../06_开发规范/Saga使用指南.md)`

❌ **错误**: 在多个文档中重复完整的 Saga 说明

### 2. 轻量审查机制

#### PR 级别审查

新增文档时检查：
- [ ] 必要性：能否归并到现有文档？
- [ ] 唯一性：是否与现有文档重复？
- [ ] 大小：预计行数是否 < 500 行？
- [ ] SSOT：是否声明了主题边界？

#### 自动化检查

```yaml
# GitHub Actions 示例
- 文件数量检查（> 60 则警告）
- 单文件大小检查（> 1000 行则警告）
- Legacy 文件检测（存在则失败）
```

#### 季度健康检查

- 文档总数是否超标？
- 是否有长期未更新的文档？
- 是否有新的重复内容？
