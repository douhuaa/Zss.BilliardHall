## 概述

本文件为 GitHub Copilot 提供关于新人入职文档的场景化指导。

**核心原则（来自 ADR-960）**：
- 提供清晰的入职路径
- 文档分层次、分阶段
- 便于新人快速上手
- 保持入职文档与 ADR 同步

---

## 场景 1：新人如何开始

### 用户问："新人应该从哪里开始学习？"

**Copilot 应该**：

```markdown
新人学习路径：

## 第一阶段：了解架构（第 1 天）
1. 阅读 README.md - 项目概览
2. 阅读 docs/ARCHITECTURE.md - 架构概览
3. 了解核心 ADR：
   - ADR-0001：模块化架构
   - ADR-0005：交互模型

## 第二阶段：动手实践（第 2-3 天）
1. 跟随 docs/guides/getting-started.md 搭建环境
2. 运行架构测试
3. 阅读一个模块的代码

## 第三阶段：深入学习（第 1 周）
1. 阅读更多 ADR
2. 查看实践案例 docs/cases/
3. 尝试小的代码修改
```
