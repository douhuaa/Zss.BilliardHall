## 案例文档标准

根据 ADR-951，每个案例必须包含：

```markdown
# 案例标题

**难度**：🟢 简单 / 🟡 中等 / 🔴 困难  
**相关 ADR**：[ADR-XXXX](../adr/path)  
**作者**：@username  
**日期**：YYYY-MM-DD  
**标签**：模块化, CQRS, 测试

## 适用场景

描述此案例适用的场景

## 背景

为什么需要这个案例

## 解决方案

### 架构设计

### 代码实现

### 测试验证

## 常见陷阱

列出容易犯的错误

## 参考资料

- ADR 链接
- 相关文档
```

## 审核标准

提交案例前必须确保：

- ✅ 符合相关 ADR 规范
- ✅ 代码可运行（或提供完整示例）
- ✅ 经过 Code Review
- ✅ 包含测试验证
- ✅ 更新本 README

## 如何贡献

2. 确保代码符合架构约束

4. 更新本 README 的索引
